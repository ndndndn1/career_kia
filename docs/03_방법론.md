# 03. 방법론

> 본 문서는 각 파이프라인 단계의 **의사결정 근거**를 정리한다. 구현 코드는 `src/career_kia/`에, 실행 결과는 `notebooks/`에 있다.

## 3.1 전처리

### 결측치 보간

| 상황 | 기법 | 근거 |
|---|---|---|
| 공정 파라미터 단발성 결측 | 선형 보간 | 물리량 연속성 전제 |
| 짧은 진동 신호 gap | 큐빅 스플라인 | 고주파 성분 유지 |
| 다변수 동시 결측 | KNN Imputer | 변수 간 상관 활용 |
| 계절성이 있는 시계열 | seasonal_decompose + 선형 | 주기 성분 보존 |

### 노이즈 필터링

- **Butterworth 대역통과**: 베어링 특성 주파수 주변만 추출 (예: 500–5,000 Hz)
- **Wavelet 잡음 제거**: Daubechies(db4) level 4, soft thresholding — 충격성 결함 보존에 유리
- 두 기법을 비교 시각화하여 공정/결함 유형별 최적 조합 제안

### 이상치 탐지

- 단일 변수: IQR / MAD(로버스트)
- 다변수: Isolation Forest
- 시계열: 이동창 Z-score + LOF 조합

### 멀티 센서 동기화

- CWRU 12k/48k와 AI4I 이벤트 기반 로그를 공통 타임베이스로 리샘플 (`TARGET_FS = 12k`, polyphase filter)
- 타임스탬프 기준 `merge_asof`로 센서 ↔ 공정 파라미터 조인

## 3.2 피처 엔지니어링

### 윈도잉

- `WINDOW_SIZE = 2048` (약 170 ms @ 12kHz), 50% 오버랩
- 창 단위로 라벨 전파 + 피처 산출

### 시간 도메인

- 통계: RMS, Peak-to-peak, Crest Factor, Skewness, Kurtosis
- 신호 형상: Impulse Factor, Shape Factor, Margin Factor
- 엔벨로프 기반 Kurtosis (충격성 결함 감지)

### 주파수 도메인

- FFT 스펙트럼 → 대역별 에너지 (Band Power)
- **Envelope Spectrum**: 힐버트 변환 기반 → BPFI / BPFO / BSF / FTF 특성 주파수 진폭 (베어링 결함 진단의 표준)
- 웰치 PSD → 스펙트럴 엔트로피
- 스펙트로그램 기반 STFT / 웨이블렛 스칼로그램 통계

> 위 피처 설계는 "공정 특성을 반영한 특징공학" 문구에 정면 대응. 베어링 물리(BPFI, BPFO)를 반영하는 것이 핵심.

## 3.3 모델링

### 혼합 모델링 (Performance × Interpretability)

- **LightGBM**: 비선형/상호작용 포착, 성능 지향
- **pyGAM**: 변수별 비선형 스무딩 성분으로 해석 용이
- 스태킹: LightGBM 예측 확률 + 원 피처 → pyGAM 메타 모델

```
LightGBM ──┐
           ├── → pyGAM (메타) ─→ 최종 예측
원 피처 ───┘
```

이 구조로 성능(ROC-AUC, F1)과 해석(각 변수의 기여 형태)을 함께 확보한다.

### 베이스라인

- Logistic Regression (L2): 선형 기준점
- Random Forest: 비선형 기준점

### 평가

- Stratified K-Fold (5), Batch leakage 방지 위해 `GroupKFold(batch_id)` 사용
- 지표: ROC-AUC, PR-AUC, F1, Precision@Recall=0.9

## 3.4 XAI

### SHAP

- TreeExplainer (LightGBM) — 빠르고 정확
- Global: Summary/Beeswarm, Mean |SHAP|, Interaction 값
- Local: Waterfall, Force plot
- **시점별**: 시간축으로 샘플을 정렬하고 SHAP 값의 이동평균 트렌드 산출 → 리스크 변화 시점 식별

### LIME

- Tabular 샘플링 기반 로컬 설명
- SHAP과 교차 검증 관점으로 사용

### 설명 템플릿 (표준화)

`xai/explanation_templates.py`에서 다음 포맷을 표준화한다.

1. **변수 기여도** — 상위 K개 변수와 방향성
2. **임계 구간** — PDP/ICE 기반 위험 급증 구간
3. **상호작용 효과** — SHAP interaction 상위 쌍

### 자연어 생성

`xai/nl_generator.py` — SHAP 결과를 규칙 기반으로 한국어 문장으로 변환.

예시: "이 배치는 회전수 2100rpm 이상에서 토크가 58Nm을 초과했을 때 주요 리스크가 발생했습니다. 공구 마모 200분 구간과 결합 시 불량 확률이 크게 증가하는 상호작용이 관찰됩니다."

## 3.5 인과추론

### DAG 구성

도메인 지식 기반 초기 DAG:

```
Type(품질등급) ─→ Process temp ─┐
Tool wear ──────→ Torque ───────┼─→ Machine failure
Rotational speed ─→ Process temp ─┤
Air temp ──────────────────────────┘
```

- `causal-learn`의 PC / GES를 보조적으로 써서 관측 기반 구조를 교차 검증
- 교란변수: `Type`, `Tool wear`는 대표적인 confounder로 통제

### 개입 효과 추정

- DoWhy 4단계: Model → Identify → Estimate → Refute
- 추정기: Backdoor + Propensity Score Matching, Linear Regression, Double ML(EconML)
- Refutation: Random Common Cause, Placebo Treatment, Data Subset

### 시계열 인과

- Granger Causality: 공정 파라미터와 진동 특성 간 선행성 검토
- PCMCI (tigramite) 검토 코드 — 시변 교란변수 완화

## 3.6 대시보드 & MLOps

- **MLflow**: 실험 파라미터·지표·모델 자동 로깅, 모델 레지스트리로 버전 관리
