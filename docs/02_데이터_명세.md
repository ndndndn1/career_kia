# 02. 데이터 명세

## 2.1 데이터셋 개요

실제 자동차 제조 현장 데이터는 기밀·IP 보호로 공개본이 제한적이므로, 공개 데이터셋을 **가상의 자동차 파워트레인 가공 라인** 시나리오에 매핑하여 사용한다. 방법론은 그대로 실제 현장에 이식 가능한 형태로 구현한다.

### A. 진동 신호 — CWRU Bearing Dataset

- 제공: Case Western Reserve University Bearing Data Center
- 샘플링 주파수: 12 kHz / 48 kHz
- 신호 채널: 드라이브엔드(DE), 팬엔드(FE), 베이스(BA) 가속도계
- 결함 유형: 정상(Normal), Inner Race(IR), Outer Race(OR), Ball(Ball)
- 결함 크기: 0.007 / 0.014 / 0.021 inch
- 운전 조건: 0~3 HP 부하, 1797~1720 rpm

본 프로젝트에서는 이를 **스핀들 베어링 가속도 신호**로 매핑한다.

### B. 공정 파라미터 — UCI AI4I 2020 Predictive Maintenance

- 제공: UCI ML Repository (Matzka, CC BY 4.0)
- 행 수: 10,000
- 피처:
  - `Air temperature [K]`, `Process temperature [K]`
  - `Rotational speed [rpm]`, `Torque [Nm]`
  - `Tool wear [min]`, `Type` (L/M/H 품질 등급)
- 라벨:
  - `Machine failure` (이진)
  - 5개 failure mode: TWF, HDF, PWF, OSF, RNF

본 프로젝트에서는 이를 **CNC 가공 공정 파라미터와 기계 고장 라벨**로 매핑한다.

### C. 합성 MES 메타 (본 프로젝트 생성)

두 레이어를 같은 가상 생산라인 관점으로 연결하기 위해 다음 필드를 합성한다.

| 필드 | 설명 |
|---|---|
| `batch_id` | 10분 단위 생산 배치 식별자 |
| `timestamp` | 배치 시작 시각 (UTC) |
| `shift` | A/B/C 3교대 |
| `operator_id` | 10명의 가상 작업자 |
| `line_id` | PL-01 / PL-02 두 라인 |

합성 규칙은 `src/career_kia/data/loaders.py`에 공개하여 완전한 재현성을 보장한다.

## 2.2 가공 단계별 데이터 경로

| 단계 | 경로 | 설명 |
|---|---|---|
| 원본 | `data/raw/cwru/`, `data/raw/ai4i/` | 다운로드 그대로 |
| 중간 | `data/interim/` | batch 매핑, 리샘플링 완료 |
| 가공 | `data/processed/` | 피처 테이블(parquet) |

원본 데이터는 저장소에 커밋하지 않으며 `make data` 명령으로 자동 다운로드된다.

## 2.3 라이선스 및 사용 조건

- CWRU: 비상업 연구·교육 목적. 논문 인용 권장.
- AI4I 2020: CC BY 4.0 (Matzka, S. 2020, "Explainable Artificial Intelligence for Predictive Maintenance Applications").
- 본 프로젝트: 개인 포트폴리오 용도. 코드는 자유롭게 참고 가능.
