# 01. 프로젝트 개요

## 1.1 배경

Software Defined Factory(SDF)는 공장 운영을 소프트웨어 중심으로 재구성하여, 센서·설비·MES 데이터를 실시간으로 수집·분석하고 의사결정에 반영하는 제조 패러다임이다. 본 프로젝트는 SDF 관점의 제조 AI 솔루션에 필요한 핵심 역량 네 가지 — **다중 센서 전처리**, **불량/고장 원인 분석**, **설명 가능 AI(XAI)**, **인과추론** — 을 한 편의 end-to-end 파이프라인으로 엮어 참조 구현한다.

## 1.2 문제 정의

자동차 파워트레인 부품 가공 라인(샤프트, 기어 등)에서 발생하는 품질 저하는 단일 인자가 아닌 **스핀들/설비 상태(진동)** 와 **CNC 공정 조건(회전수·토크·공구 마모 등)** 의 상호작용으로 결정된다. 현장 엔지니어가 실제 조치를 취하려면 다음이 필요하다.

1. 조기 예측 (어느 배치/시점에 위험이 있는가)
2. 설명 (왜 그런 판단을 했는가, 어느 변수가 기여했는가)
3. 처방 (무엇을 바꾸면 불량률이 줄어드는가 — 인과)

## 1.3 접근 방법 요약

| 단계 | 산출물 |
|---|---|
| 데이터 | CWRU 베어링 진동 + AI4I 공정 파라미터 → 합성 batch로 조인 |
| 전처리 | 결측 보간 / 노이즈 필터링(Butterworth, Wavelet) / 이상치 탐지 / 멀티레이트 동기화 |
| 피처 | 윈도잉 + 시간/주파수 도메인 피처 수십 종 |
| 모델 | LightGBM(성능) + pyGAM(해석) 하이브리드 스태킹 |
| XAI | SHAP/LIME + 설명 템플릿 + 자연어 설명 |
| 인과 | 도메인 DAG + DoWhy ATE/CATE + 시계열 인과(PCMCI) 검토 |
| 배포 | Streamlit 4페이지 대시보드 + MLflow 실험 추적 |
